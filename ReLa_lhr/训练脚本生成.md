# RevealLayer 训练脚本生成

## 目标

在 `E:\Image_Decomposition\code\RevealLayer` 中，基于现有推理代码、`RevealLayer.pdf` 和外部训练参考，复现一套可训练的 LoRA 训练脚本。训练实现必须贴近论文描述，同时尊重仓库已有模型、pipeline、数据格式和推理路径。

默认只生成或修改训练脚本、配置和说明文档；除非用户明确要求，不要启动真实训练。

## 核心原则

- 先读代码和论文，再写训练代码。不要只套通用 Diffusers/Flux LoRA 模板。
- RevealLayer 的推理路径是任务锚点：bbox/query、layer 数量、`[full image, background, foreground layers...]`、latent shape、`latent_image_ids`、ROI attention、OGA/refiner 和透明 VAE 接口必须对齐。
- DiffSynth FLUX 是训练机制锚点：FlowMatch timestep、scheduler target、training weight、LoRA 注入、checkpoint/logging/launcher 的设计要优先参考它。
- Qwen-Image-Layered 是图层任务锚点：参考其 RGBA latent、variable layers、multi-layer decomposition objective 和数据 operator 思路，但不要照搬其 Qwen-Image pipeline。
- 如果论文、推理代码、DiffSynth 训练方式和 Qwen-Image-Layered 参考之间冲突，先写清楚冲突，再选择与 RevealLayer 推理代码最兼容的最小实现。
- 不要臆造论文没有支持的 loss 或模块。缺失信息必须在代码注释和训练文档中标注为“工程近似”或“待论文/作者代码确认”。

## 已知论文设置

从 `RevealLayer.pdf` 提取训练设置时，应优先核对以下信息：

- 基模：FLUX.1-dev。
- 微调方式：LoRA。
- LoRA rank：64。
- 优化器：Prodigy。
- 学习率：1.0。
- 训练步数：50,000 iterations。
- 全局 batch size：8。
- 输入图像长边 resize 到 1024。
- 总 loss：`L = L_FM + lambda_alpha L_alpha + lambda_o L_orth`，其中 `lambda_alpha = 1.0`、`lambda_o = 1.0`。
- `L_alpha` 来自透明/alpha 监督；`L_orth` 用于抑制背景重建中的前景残留。实现前必须确认透明 decoder 和像素空间目标是否在当前代码/data 中可用。

从 `Qwen-Image_Layered.pdf` 提取参考时，关注：

- 输入 RGB 图像分解为多个 RGBA layers。
- 使用 Flow Matching 预测 target velocity。
- 输出 RGBA layers 独立编码，不沿 layer 维压缩。
- 使用 variable-layer MMDiT 思路支持可变层数。
- 多阶段训练和 max layers 是任务设计参考，不是 RevealLayer 的直接训练配置。

## 必须对照的 DiffSynth 训练机制

实现或审查训练脚本时，必须核对这些机制：

- `FluxTrainingModule.get_pipeline_inputs()` 如何构造 `input_image`、`height`、`width`、`cfg_scale=1`、`embedded_guidance=1`、`t5_sequence_length=512`、`rand_device` 和 gradient checkpointing 输入。
- `DiffusionTrainingModule.switch_pipe_to_training_mode()` 如何：
  - 调用 `pipe.scheduler.set_timesteps(1000, training=True)`。
  - freeze 非训练模块。
  - 通过 `inject_adapter_in_model` 注入 LoRA。
  - 加载 LoRA checkpoint。
- `FlowMatchSFTLoss()` 如何：
  - 从 scheduler timesteps 中采样 timestep。
  - 用 `scheduler.add_noise(input_latents, noise, timestep)` 构造 noisy latents。
  - 用 `scheduler.training_target(input_latents, noise, timestep)` 得到训练目标。
  - 用 `scheduler.training_weight(timestep)` 加权 loss。
- `flow_match.py` 中 FLUX scheduler 的 target convention。不要手写与 scheduler convention 冲突的 velocity 公式，除非明确说明与 RevealLayer 论文公式一致且已验证底模输出符号。
- DiffSynth 默认 LoRA target 是较保守的核心 DiT 模块，例如 `q,k,v,o,ffn.0,ffn.2`。RevealLayer 可按模型命名映射到自定义 FLUX transformer，但不要默认把所有 embedding/proj/refiner 都混进 LoRA。

## Qwen-Image-Layered 参考边界

读取 `E:\Image_Decomposition\code\Qwen_Image_Layered\scripts\train.py` 时，关注：

- `UnifiedDataset` + `special_operator_map` 如何处理 `layer_input_image`、RGBA image list、`context_image`。
- 图层输入通过 `LoadImage(convert_RGB=False, convert_RGBA=True)` 和 crop/resize 对齐。
- `FlowMatchSFTLoss` 仍复用 DiffSynth 的训练模块。
- `zero_cond_t`、processor、Qwen tokenizer、Qwen pipeline 是 Qwen-Image-Edit/Qwen-Image 专属逻辑，不应搬到 RevealLayer。

读取 `Qwen-Image_Layered.pdf` 时，关注：

- RGBA layer decomposition 的训练目标与 layer 表示。
- 可变层数和 layer 维建模的设计动机。
- 图层组合/alpha blending 作为数据和验证语义参考。

## 实现流程

1. 建立论文、推理代码、DiffSynth FLUX 训练、Qwen-Image-Layered 参考的四列表。
   - 列出训练模块、loss、scheduler/timestep、LoRA target、数据字段、保存格式。
   - 标明每项来自哪个文件/论文。
   - 对无法直接对应的部分，写清楚“新增”“近似实现”“暂不实现”。

2. 设计数据集接口。
   - 数据加载必须和 RevealLayer 推理脚本输入语义对齐。
   - 支持 manifest/JSONL/JSON，字段至少覆盖原图、prompt、bbox 列表、目标 RGBA layers、背景或合成图。
   - 对齐 `infer.py` 的 bbox 缩放、裁剪和 16 像素对齐逻辑。
   - 保留 `[full image, background, foreground layers...]` 顺序。
   - Windows/PowerShell 导出的 JSON 可能带 BOM，读取 JSON 时优先使用 `encoding="utf-8-sig"`。
   - 如果借鉴 Qwen-Image-Layered 的 operator 思路，只借鉴 RGBA/list/context 的数据处理方式，不引入 Qwen pipeline 依赖。

3. 设计训练前向。
   - 复用 RevealLayer 推理 pipeline 中的模型加载、图像预处理、bbox/query 编码、`latent_image_ids`、adapter data、ROI mask 和 refiner 调用。
   - 参考 DiffSynth 将训练流程拆成：输入准备 -> prompt/image/latent 编码 -> scheduler 加噪 -> model_fn/transformer 前向 -> scheduler target loss。
   - 对 full image 条件层要特别处理：如果 layer 0 是条件输入而非预测目标，不要让它参与 FM loss。
   - 当前自定义 `list_layer_box` 若未 batch 化，优先使用 `train_batch_size=1` 加 `gradient_accumulation_steps` 实现全局 batch，而不是草率改 transformer batch 语义。

4. 设计 loss。
   - `L_FM` 必须和 FLUX scheduler 的 prediction type、timestep 尺度和 target 符号一致。
   - 优先参考 DiffSynth 的 `FlowMatchSFTLoss`：`scheduler.add_noise`、`scheduler.training_target`、`scheduler.training_weight`。
   - 若使用 RevealLayer 论文公式 `z_t = t z0 + (1-t) z1`、`v = z0 - z1`，必须说明它和 DiffSynth convention 的关系，并验证 transformer timestep 输入尺度。
   - `L_alpha` 和 `L_orth` 只有在透明 decoder、RGBA/alpha target 和像素空间预测链路可用时才接入；否则配置项应明确禁用或报错，不能静默无效。
   - 对背景、前景和 full image 条件层的 loss mask/weight 必须明确，不要默认所有 layer 等权。

5. 设计 LoRA 注入。
   - 先冻结基础模型，只打开 LoRA 或论文明确需要训练的模块。
   - LoRA 注入方式优先参考 DiffSynth 的 PEFT 注入方式和 target module 选择。
   - 默认 target 应保守：attention q/k/v/o 与 FFN linear；只有在验证需要时再加入 `x_embedder`、`context_embedder`、`proj_out`、`layer_pe`、`refiner`。
   - 若训练 `layer_pe` 或 `refiner`，应与 LoRA 权重分开保存，并说明推理加载方式。
   - 保存的 `pytorch_lora_weights.safetensors` 必须能被 `pipeline.load_lora_weights()` 或对应推理 loader 加载。

6. 加入训练基础设施。
   - 使用 `accelerate` 支持单卡/多卡、混合精度、梯度累积、断点恢复。
   - 支持 TensorBoard loss log。
   - 支持 checkpoint 保存、latest resume、指定 checkpoint resume。
   - 支持 `gradient_checkpointing`、可选 offload、PyTorch SDPA/xformers 等训练加速开关，按环境可用性降级。
   - 支持 seed、dataloader workers、batch size、learning rate、warmup、训练步数、保存间隔、验证间隔等 CLI 参数。

7. 补充训练文档。
   - 写清数据格式、路径布局、启动命令、多卡命令、恢复训练命令、日志位置、权重保存位置。
   - 单独列出哪些细节来自 RevealLayer 论文、哪些来自 DiffSynth FLUX、哪些借鉴 Qwen-Image-Layered、哪些是工程近似。
   - 给出最小 smoke test 命令，但不要默认运行长时间训练。

## 验收标准

完成后至少验证：

- `python -m py_compile` 通过新增/修改的 Python 文件。
- 训练脚本 `--help` 可运行，且不需要完整大模型权重。
- 数据集能用小 manifest 初始化并返回 batch，shape、dtype、路径解析、bbox 数量和 layer 顺序符合预期。
- LoRA 注入后只有预期参数 `requires_grad=True`。
- `L_FM` 的 noisy latent、target velocity、timestep 尺度、loss mask 与 DiffSynth/RevealLayer 约定一致。
- full image 条件层不被误当作预测目标训练，除非文档明确说明这样做的原因。
- checkpoint、TensorBoard log、输出目录参数完整。
- 文档中的命令和脚本参数一致。

如用户明确要求 smoke test，可以只跑极小步数，例如 1 batch / 1 step，并避免触发完整训练。

## 需要特别报告的风险

- RevealLayer 论文训练细节缺失，导致只能实现近似版本。
- DiffSynth scheduler convention 与 RevealLayer 论文公式存在符号或 timestep 参数化差异。
- 当前仓库缺少训练所需的目标 RGBA layer、alpha、背景或 metadata。
- 模型自定义组件和标准 Diffusers/PEFT LoRA 接口不完全兼容。
- 推理权重保存格式与训练 LoRA 权重格式需要额外适配。
- 自定义 transformer 尚未支持 batched `list_layer_box`。
- 训练显存需求过高，需要降低分辨率、batch size、layer 数或启用 checkpointing/offload。
